EXECUTIVE SUMMARY (≤150 lines, minimal grammar)

RECOMMENDATION: React 19 + Vite + Tailwind + shadcn/ui + ECharts + React Hook Form + Zustand + TanStack Query

WHY THIS STACK:
- Battle-tested by Fortune 500 companies (Vercel, Shopify, Stripe, Airbnb)
- Minimal boilerplate vs Redux (Zustand 2.4 KB vs 50 KB)
- Quarkus integration proven + Quinoa extension simplifies bundling
- <200 KB gzip total (competitive, lean bundle)
- Fast dev experience: Vite <1s HMR, Quarkus hot reload
- TypeScript + Zod provide type safety end-to-end
- shadcn/ui copy-paste components = full customization flexibility

================================================================================

COMPONENT BREAKDOWN:

Frontend Framework:  React 19 (45 KB)
  → Industry standard, largest ecosystem, proven Quarkus patterns

Build Tool:          Vite
  → <1 second HMR, npm-native ESM, faster than webpack

Styling:             Tailwind CSS v4 (18 KB)
  → Utility-first, zero runtime, production CSS purged

Components:          shadcn/ui (copy-paste, headless)
  → Full customization, zero lock-in, Radix UI primitives

Routing:             React Router v6
  → Standard practice, code-splitting per route

Forms:               React Hook Form (8.6 KB) + Zod (5 KB)
  → Uncontrolled = minimal re-renders, zero deps, type-safe schemas

Data Visualization:  ECharts + echarts-for-react
  → 30+ chart types, handles millions of data points, professional

State Management:    Zustand (2.4 KB)
  → App state only, separates concerns from API caching

API Caching:         TanStack Query (React Query) (45 KB)
  → Built-in: caching, background sync, optimistic updates, deduplication

HTTP Client:         Axios
  → Standard practice, interceptors for auth tokens

Backend:             Quarkus (Java)
  → REST APIs, OpenAPI docs, JWT auth, CORS configured

Integration:         Quarkus Quinoa
  → Embeds React dist in JAR, single deployment artifact

QUARKUS INTEGRATION (SIMPLIFIED):

1. Add dependency: io.quarkiverse:quarkus-quinoa:3.2.0
2. Config: Point to React dist folder
3. Build: npm run build → mvn clean package
4. Output: Single JAR with embedded React
5. Deploy: One artifact (no separate frontend)
6. Serve: / (React) + /api/* (REST endpoints)

Benefits:
- Zero CORS complexity in production
- Simplified DevOps (one version management)
- Single deployment artifact
- No separate frontend server needed

================================================================================

DEV WORKFLOW:

Terminal 1:  npm run dev (React Vite, :5173, HMR enabled)
Terminal 2:  mvn quarkus:dev (Quarkus, :8080, hot reload Java)

Frontend automatically proxies API calls to Quarkus backend.
